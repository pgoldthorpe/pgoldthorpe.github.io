Creating a github.io pages blog using Pelican

Fri 21 August 2020

Introduction

It seems only fitting that my first post is a meta-post about the creation of the blog.

I wanted to create a blog. The idea was to have a central repository for my interests and a place to make a protfolio and place my work. To setup this blog I knew a few things. 1. I wanted to host the blog with GitHub pages 2. I wanted minimal effort to keep it running I looked at a few static site generators. Gatsby, Hugo, Jekyll, Pelican. Oof. There are so many tools out there and it's easy to get overwhelmed. I learn best from example. One of the major influences was a blog by Jake Vanderplas at Pythonic Perambulations. If it's good enough for a Software Engineer at Google who taught me Python, it's good enough for me! Pelican is also written in Python which is a language known for its simplicity and accessibility. It also helped that I have Python already installed on my computer.

I had already setup a github pages on my GitHub account so it would be a simple matter of placing the files in that repo. I created a virtual environment and installed pelican. It was great. pelican-quickstart asked a few questions for creation of the blog and then... it was done.

The way to make content is by creating new Markdown files in the content folder. To generate the site, it's one simple pelican command in the terminal. This was easier than I expected to get up and running. Now I have no excuses.
